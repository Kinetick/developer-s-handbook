# Куча (Heap)

[🠔 Оглавление](../index.md#навигация-по-разделам)

Куча — это структура данных, единственной целью которой является гарантия одного из инвариантов:

1. Родитель **всегда** больше или равен своему потомку — **Max-Heap**.
2. Родитель **всегда** меньше или равен своему потомку — **Min-Heap**.

Если куча гарантирует выполнение первого пункта, то на вершине кучи **всегда** будет максимальный по значению элемент. Если второго — **всегда** будет минимальный по значению элемент.

---

## Ключевые особенности

1. **Свойство кучи**: Каждый родительский узел удовлетворяет инварианту относительно своих потомков.
2. **Полное бинарное дерево**: Куча всегда представляет собой полное бинарное дерево, где все уровни полностью заполнены, кроме, возможно, последнего, который заполняется слева направо.
3. **Представление в массиве**: Куча хранится в виде массива, где вершина дерева — это первый элемент массива (индекс `0`).

> **Важно**: Куча **не гарантирует** упорядоченность элементов на одном уровне или между ветвями. Она гарантирует только отношение родитель-потомок.

---

## Представление в массиве

Кучу можно хранить в виде **плоского бинарного дерева** — массива. Индексы соседей вычисляются математически:

```text
Индекс левого потомка  = 2 * индекс текущего элемента + 1
Индекс правого потомка = 2 * индекс текущего элемента + 2
Индекс родителя       = (индекс текущего элемента - 1) // 2
```

---

## Операция просеивания

Важнейшей операцией кучи является **просеивание** (heapify) — элементы перемещаются внутри массива для обеспечения выполнения инварианта.

### Построение кучи из массива

Для построения кучи из произвольного массива используется алгоритм **Floyd's heap construction**:

1. Находится последний нелистовой узел по формуле: `(n // 2) - 1`, где `n` — количество элементов.
2. Начиная с этого узла, происходит обход массива **снизу вверх** до корня.
3. Для каждого узла выполняется просеивание вниз (`sift-down`), восстанавливающее инвариант кучи.

> **Пример**: Для массива длиной 10 элементов (индексы `0–9`) последний нелистовой узел находится по индексу `(10 // 2) - 1 = 4`. Его левый потомок: `2 * 4 + 1 = 9`, правый: `2 * 4 + 2 = 10` (выход за границы). Это означает, что начиная с индекса `4` мы охватываем все **внутренние узлы**, а элементы с индексами `5–9` являются **листьями** и не требуют просеивания.

### Почему сложность `O(n)`, а не `O(n log n)`?

Хотя просеивание одного узла занимает `O(log n)`, общая сложность построения кучи составляет `O(n)`. Это связано с тем, что:

- Большинство узлов — **листья** (примерно `n/2`), которые не требуют просеивания.
- Узлы на верхних уровнях, хотя и просеиваются на большую глубину, их количество мало.
- Математически: `Σ (высота узла * количество узлов на этой высоте) = O(n)`.

## Реализация в CPython

В Python куча представлена в виде **плоского бинарного дерева** и хранится в виде списка (массива) указателей. Модуль `heapq` реализует инвариант **Min-Heap**.

### Инвертирование кучи

Для трансформации кучи в **Max-Heap** требуется умножать элементы на `-1` перед добавлением и после извлечения:

```python
import heapq

# Max-Heap через инверсию
heap = []
heapq.heappush(heap, -value)   # добавление
max_value = -heapq.heappop(heap)  # извлечение
```

> **Важно**: В Python 3.13+ появился параметр `reverse` в функции `heapify()`, позволяющий создавать **Max-Heap** без явной инверсии значений.

### Извлечение вершины

Поскольку вершина кучи — это первый элемент массива, его прямое удаление заняло бы `O(n)` из-за сдвига всех остальных элементов. Разработчики Python использовали следующий подход:

1. Вершина записывается во временный буфер.
2. Последний элемент массива перемещается в начало.
3. Последний элемент удаляется за `O(1)`.
4. Запускается процесс просеивания вниз (`sift-down`) для восстановления инварианта.

## Сложность операций

Ниже приведена оценка временной и пространственной сложности основных операций.
*Формат: `Время (средний/худший случай) | Пространство`*

| Операция | Метод / Синтаксис | Временная сложность | Пространственная сложность | Примечание |
| --- | --- | --- | --- | --- |
| **Создание кучи из списка** | `heapq.heapify()` | `O(n)` | `O(1)` | Алгоритм Флойда: построение снизу вверх от последнего нелистового узла. |
| **Добавление элемента** | `heapq.heappush()` | `O(log n)` | `O(1)` | Элемент добавляется в конец, затем просеивается вверх (`sift-up`). |
| **Извлечение вершины** | `heapq.heappop()` | `O(log n)` | `O(1)` | Последний элемент заменяет вершину, затем просеивается вниз (`sift-down`). |
| **Просмотр вершины** | `heap[0]` | `O(1)` | `O(1)` | Доступ к первому элементу массива. |
| **Создание поэлементно** | итеративный `heappush()` | `O(n log n)` | `O(n)` | Последовательное добавление элементов с просеиванием каждого. |
| **Замена вершины** | `heapq.heapreplace()` | `O(log n)` | `O(1)` | Извлечение вершины и добавление нового элемента за одну операцию. |
| **Слияние двух куч** | `heapq.merge()` | `O(n log k)` | `O(k)` | Возвращает итератор, не создает новую кучу в памяти. |

## Min-Heap vs Max-Heap

| Характеристика | Min-Heap | Max-Heap |
| --- | --- | --- |
| **Инвариант** | Родитель ≤ Потомок | Родитель ≥ Потомок |
| **Вершина** | Минимальный элемент | Максимальный элемент |
| **Реализация в Python** | `heapq` (по умолчанию) | Инверсия значений или `heapq._heapify_max()` (внутренний API) |
| **Применение** | Очерти с приоритетами, алгоритм Дейкстры | Нахождение топ-K максимальных элементов |

> **Примечание**: Функция `heapq._heapify_max()` является внутренней и не гарантируется в будущих версиях Python. Для production-кода рекомендуется использовать инверсию значений или параметр `reverse` (Python 3.13+).
