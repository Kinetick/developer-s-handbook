# Массив

**Массив** — это структура данных, для которой характерны следующие особенности:

## Ключевые особенности

1. **Непрерывная область в памяти** (в отличие от связанного списка).
2. Элементы массива **всегда одного типа**.
3. **Мгновенное получение** элемента по его индексу.
4. **Мгновенная замена** значения элемента по индексу.

> Мгновенные операции получения и изменения элемента по индексу вытекают из пунктов 1 и 2. В языках, подобных C, в переменной массива хранится указатель на самый первый элемент с индексом `0`. Если область памяти непрерывна и тип всех элементов одинаков, то переход к нужному участку памяти через арифметику указателей — это дешевая операция.

---

## Реализация в CPython

В **Python** список — это массив указателей на значения в памяти. Это важно понимать: массив хранит **указатели** (адреса в памяти), а не сами значения. Именно поэтому список способен хранить разнородные структуры внутри себя.

### Структура данных

- В ячейках массива хранятся указатели на объекты Python в куче памяти.
- Индекс ячейки определяется арифметикой указателей.
- Физически значения располагаются в куче памяти, но благодаря известному адресу можно мгновенно получить доступ к любому элементу.

### Арифметика указателей

Например, есть массив из 10 элементов типа `int8_t` (1 байт). Для получения пятого элемента (индекс `4`) нужно вычислить его адрес в памяти по формуле:

```txt
адрес = указатель_на_начало_массива + 4 * размер(int8_t)
```

### Цена за скорость

За высокую скорость доступа к элементам приходится платить: **любая операция удаления элемента, кроме удаления из конца массива, требует дефрагментации памяти**.

Если удалить элемент из середины массива, потребуется:

1. Сдвинуть **все** элементы после удаляемого влево на одну позицию.
2. Усечь конец массива.

### Избыточное выделение памяти

Раз список — это по сути массив, значит он имеет все преимущества последнего и разделяет его недостатки. Чтобы скомпенсировать проблемы массивов из C, разработчики Python используют оптимизацию — **избыточное выделение памяти**.

Эта оптимизация нужна для амортизации выделения памяти при добавлении элемента в конец списка. Избыточное выделение рассчитывается динамически по формуле CPython:

| Размер списка | Оверхед памяти |
| --- | --- |
| Малые объемы | ~30–60% |
| Большие объемы | ~12.5% |

Это позволяет эффективно балансировать между скоростью добавления и экономией RAM.

## Сложность операций

Ниже приведена оценка временной и пространственной сложности основных операций.
*Формат: `Время | Пространство`*

> - **Реальный размер массива** — `size`.
> - **Выделенная память** в непрерывном блоке под указатели — `allocated`.

| Операция | Метод / Синтаксис | Временная сложность | Пространственная сложность | Примечание |
| --- | --- | --- | --- | --- |
| **Добавление в конец** | `append()` | `O(1)` | `O(1)` | В худшем случае, если `size >= allocated`, временная сложность вырастет до `O(n)` за счет тяжелой функции `realloc`. |
| **Удаление с конца** | `pop()` | `O(1)` | `O(1)` | Освободившийся слот памяти не возвращается в систему, а резервируется в качестве амортизатора. По умолчанию `pop()` удаляет последний элемент. |
| **Добавление в произвольное место** | `insert()` | `O(n)` | `O(1)` | Худший вариант — вставка в голову списка. Придется сдвинуть все элементы в позиции `[1:]` на один индекс вправо. |
| **Удаление из произвольного места** | `pop(index)` | `O(n)` | `O(1)` | Более общий случай удаления из головы списка. |
| **Расширение** | `extend()` | `O(k)` | `O(k)` | Если `size < allocated`, дополнительное выделение памяти не требуется. Если `size >= allocated`, временная сложность может возрасти до `O(n+k)` из-за `realloc`. |
| **Итерация** | `for x in list` | `O(n)` | `O(1)` | Перебор всех элементов массива. |
| **Получение среза** | `list[x:y]` | `O(k)` | `O(k)` | Переход на первый элемент среза — `O(1)`, затем итерация `k` раз — `O(k)` и выделение памяти под новый список — `O(k)`. |
| **Вставка среза** | `list[x:y] = other[x1:y1]` | `O(n+k)` | `O(n+k)` | Переход в место вставки — `O(1)`, копирование новых элементов — `O(k)`, последующий сдвиг — `O(n)` в худшем случае. Амортизация может сгладить пространственную сложность при маленьком срезе до `O(1)`. |
| **Удаление среза** | `del list[0:x]` | `O(n)` | `O(1)` | Худший сценарий — удаление среза с головы списка. Придется двигать все элементы справа налево. |
| **Умножение** | `l * k` | `O(nk)` | `O(nk)` | Является частным случаем расширения списка. |
| **Сортировка** | `sort()` | `O(n log n)` | `O(1)` | В лучшем случае (уже отсортированный список) Timsort обнаруживает существующие вхождения и отработает за `O(n)` временной сложности и `O(1)` пространственной. В худшем случае потребуется дополнительная память для хранения подсписков для слияния, и пространственная сложность возрастет до `O(n)`. |
| **Получение по индексу** | `list[x]` | `O(1)` | `O(1)` | Арифметика указателей. |
| **Проверка на вхождение** | `x in list` | `O(n)` | `O(1)` | Физически придется пройтись по всем элементам. |
| **Копирование** | `copy()` | `O(n)` | `O(n)` | Физически придется пройтись по всем элементам и скопировать их. Важно учесть, что это **поверхностное копирование** — сами данные в куче не копируются. |
| **Поиск минимума/максимума** | `min()`, `max()` | `O(n)` | `O(1)` | Физически придется пройтись по всем элементам. |
| **Получение длины** | `len()` | `O(1)` | `O(1)` | Длина списка хранится в отдельной структуре метаданных и всегда актуальна, поэтому не считается каждый раз. |

## Кортеж vs Список

Физически кортеж — это тоже массив указателей на объекты в куче, разделяющий со списками `O(1)` на чтение по индексу. Однако за счет неизменяемости (*immutability*) кортежи оптимизированы на уровне ядра CPython:

1. **Отсутствие избыточной памяти**:
   Лимит зарезервированных слотов всегда равен длине.

2. **Inline-аллокация**:
   Метаданные и указатели на элементы лежат в одном непрерывном блоке C-структуры.

3. **Free Lists (Списки свободных объектов)**:
   Уничтоженные кортежи (содержащие до 20 элементов) не всегда сразу удаляются из памяти. Они кэшируются во внутренних списках свободных объектов для мгновенного переиспользования при создании новых кортежей, что снижает нагрузку на аллокатор памяти.

4. **Мгновенное копирование**:
   При копировании кортежа Python часто не создает глубокую копию данных, а просто возвращает ссылку на существующий объект, увеличивая счетчик ссылок.
   - Время: `O(1)`
   - Память: `O(1)`
