# Бинарное дерево

**Бинарное дерево** — это топологическая иерархическая структура данных, для которой выполняется инвариант: у каждого узла не более двух потомков. Бинарное дерево является частным случаем ациклического ориентированного графа (DAG), в котором каждый узел (кроме корня) имеет ровно одного родителя. Из этого утверждения следует, что в бинарном дереве отсутствуют циклы.

---

## Ключевые особенности

Основными ключевыми понятиями бинарного дерева являются:

- **Корень** — самый верхний узел, не имеющий родителя.
- **Листья** — узлы, не имеющие потомков.
- **Уровень/глубина/высота** — высота бинарного дерева определяется как расстояние (количество ребер) от корня до самого глубокого листа.
- **Сбалансированность** — свойство, при котором высота левого и правого поддеревьев для любого узла отличается не более чем на единицу. Дерево, не удовлетворяющее этому условию, называется **несбалансированным**.
- **Заполненность** — мера плотности распределения узлов:
  - Если у каждого узла **0** или **2** потомка, такое дерево называется **строгим** (или полным в англоязычной литературе — *full*).
  - Если все узлы имеют ровно **2** потомка, а все листья находятся на одном уровне, такое дерево называется **совершенным** (*perfect*).
  - Если все уровни, кроме последнего, заполнены, а последний уровень заполняется **слева направо**, такое дерево называют **почти полным** (*complete*).

Чтобы узнать максимально возможное число узлов на каждом уровне, нужно использовать формулу:

<details>

```text
максимальное число узлов на уровне h = 2 ** h
```

</details>

Чтобы узнать высоту бинарного дерева, зная количество элементов, нужно использовать формулу:

<details>

```text
высота бинарного дерева = floor(log2(количество элементов))
```

</details>

Бинарное дерево можно превратить в плоскую структуру, например, обойти его в ширину и составить массив узлов. В этом случае математически можно вычислить индексы потомков через текущий индекс узла-родителя ([см. кучу](./heap.md#построение-кучи-из-массива)).

## Математические свойства

### Разграничение терминов

Важно различать понятия **глубины** и **высоты**:

- **Глубина узла** — это расстояние от корня до данного узла. Глубина корня равна **0**.
- **Высота узла** — это расстояние от данного узла до самого глубокого листа в его поддереве. Высота листьев равна **0**.
- **Высота дерева** — это высота его корня.

### Формулы индексов (Массивное представление)

При представлении бинарного дерева в виде массива (например, в куче) индексы узлов связаны между собой следующими формулами:

<details>

```text
# Индексы детей для узла с индексом i
left_child_index  = 2 * i + 1
right_child_index = 2 * i + 2

# Индекс родителя для узла с индексом i
parent_index = (i - 1) // 2
```

</details>

### Свойства количества узлов

Для бинарного дерева высотой $h$ (где высота корня равна 0) выполняются следующие свойства:

- **Максимальное количество узлов**: $2^{h+1} - 1$.
- **Минимальное количество узлов**: $h + 1$ (в случае вырожденного дерева, похожего на связный список).
- **Количество узлов на уровне $k$**: не более $2^k$.

## Бинарное дерево поиска (BST)

**Бинарное дерево поиска** — это подтип бинарного дерева, к инварианту которого добавляется еще одно правило: для любого узла значения в левом поддереве меньше, а в правом — больше (или равны).

Эта структура данных благодаря своему инварианту обеспечивает быстрый поиск со временной сложностью **O(log N)**. Однако для такого быстродействия необходимо, чтобы BST было сбалансировано. Если, например, добавлять значения, всегда большие предыдущего (10, 20, 30 и т. д.), произойдет вырождение BST. В этом случае оно превратится по сути в связный список, а быстрый поиск — в линейный **O(N)**.

Таким образом, для операций вставки, изменения и удаления узла появляется главная проблема — сбалансировать BST, чтобы избежать вырождения.

### Сложность операций

При рассмотрении сложности операций над BST будем опираться на сбалансированное дерево.

*Формат: `Время | Пространство`*

| Операция | Временная сложность | Пространственная сложность | Примечание |
| --- | --- | --- | --- |
| **Поиск** | `O(log N)` | `O(1)` | При сбалансированном дереве. В худшем случае (вырождение) `O(N)`. |
| **Вставка** | `O(log N)` | `O(1)` | При сбалансированном дереве. В худшем случае `O(N)`. |
| **Удаление** | `O(log N)` | `O(1)` | При сбалансированном дереве. В худшем случае `O(N)`. |
| **Минимум/Максимум** | `O(log N)` | `O(1)` | Поиск левого/правого края. |
| **Обход (In/Pre/Post)** | `O(N)` | `O(N)` | Посещение всех узлов. Пространство для стека рекурсии. |

## Реализация в Python

В **Python** нет встроенной структуры данных для реализации бинарного дерева. Традиционно эту структуру данных реализуют посредством следующего кода:

<details>

```python
import typing as t

class TreeNode:
    def __init__(self, key: str, value: t.Any) -> None:
        self.key = key
        self.value = value
        self.left: t.Self | None = None
        self.right: t.Self | None = None


root_node = TreeNode("root", 0)
a_node = TreeNode("a", 1)
b_node = TreeNode("b", 2)
c_node = TreeNode("c", 3)
d_node = TreeNode("d", 4)
e_node = TreeNode("e", 5)
f_node = TreeNode("f", 6)

root = root_node
root.left = a_node
root.right = b_node

a_node.left = c_node
a_node.right = d_node

b_node.left = e_node
b_node.right = f_node
```

</details>

## Массив vs Связные узлы

Выбор реализации бинарного дерева зависит от задачи:

- Если дерево **почти полное** и операции вставки/удаления редки, то правильный выбор — **массивное представление**. Оно экономит память на указатели и лучше использует кэш процессора.
- Если дерево **динамическое** (частые вставки и удаления) или **несбалансированное**, то лучше выбрать **связные узлы** (классы с указателями `left`/`right`). Это обеспечивает гибкость структуры.
- Если требуется **простая сериализация**, то массивное представление часто удобнее.
- Если требуется **сложная навигация** (например, доступ к родителю), то связные узлы позволяют добавить поле `parent`, что в массиве требует дополнительных вычислений.

## Области применения

Бинарные деревья широко используются в различных областях компьютерных наук:

- **Базы данных и файловые системы**: B-деревья и их вариации (основанные на принципах бинарных деревьев) используются для индексирования данных, что позволяет быстро находить, вставлять и удалять записи.
- **Парсинг и компиляторы**: Синтаксические деревья (AST) представляют структуру исходного кода или математических выражений, где каждый узел — это операция или операнд.
- **Сжатие данных**: Алгоритмы Хаффмана используют бинарные деревья для построения оптимальных кодов с переменной длиной.
- **Сетевые маршрутизаторы**: Деревья используются для эффективного поиска маршрутов в таблицах маршрутизации.
- **Игровые движки**: Деревья пространственного разбиения (например, Quadtree) помогают оптимизировать рендеринг и обнаружение столкновений.
