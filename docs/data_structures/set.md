# Множество (Set)

[🠔 Оглавление](../index.md#навигация-по-разделам)

Множество — это коллекция уникальных элементов, не сохраняющая порядок их следования.

## Ключевые особенности

1. **Уникальность**: Все элементы множества должны быть уникальными.
2. **Неупорядоченность**: Порядок элементов не гарантируется и может меняться.
3. **Математические операции**: Поддержка операций объединения, пересечения, разности и симметричной разности.

> **Важно**: Множество может содержать только **хэшируемые** (неизменяемые) объекты.
> Если бы множество позволяло хранить изменяемые объекты, изменение их состояния после вставки могло бы нарушить инвариант уникальности и сломать структуру хэш-таблицы.

---

## Реализация в CPython

Внутренне множество в Python реализовано как **хэш-таблица** (разреженный массив).

### Структура данных

- В ячейках массива хранятся указатели на объекты Python в куче памяти.
- Индекс ячейки определяется хэш-функцией от значения объекта.
- Массив называется *разреженным*, так как содержит пустые слоты для минимизации коллизий.

### Хэш-функция и коллизии

Хэш-функция — критически важный компонент реализации. В Python она детерминирована в рамках одного сеанса выполнения программы.

Для разрешения коллизий (когда разные объекты имеют одинаковый хэш) используется алгоритм **открытой адресации**. При возникновении коллизии интерпретатор последовательно ищет свободную ячейку в массиве, следуя специфическим правилам пертурбации (perturbation), а не просто линейному перебору.

## Сложность операций

Ниже приведена оценка временной и пространственной сложности основных операций.
*Формат: `Время (средний случай) | Время (худший случай) | Пространство`*

| Операция | Метод / Синтаксис | Временная сложность | Пространственная сложность | Примечание |
| --- | --- | --- | --- | --- |
| **Вставка** | `add()` | `O(1)` | `O(1)` | В худшем случае (много коллизий) время может вырасти до `O(n)`. |
| **Проверка вхождения** | `x in s` | `O(1)` | `O(1)` | Вычисление хэша и переход по индексу. |
| **Удаление** | `remove()`, `pop()` | `O(1)` | `O(1)` | Ячейка помечается как пустая, сдвиг массива не требуется. |
| **Длина** | `len()` | `O(1)` | `O(1)` | Значение кэшируется в метаструктуре объекта. |
| **Итерация** | `for x in s` | `O(n)` | `O(1)` | Перебор всех ячеек массива индексации. |
| **Копирование** | `copy()` | `O(n)` | `O(n)` | Создание нового объекта и копирование всех элементов. |
| **Объединение** | `s1 \| s2` | `O(len(s1) + len(s2))` | `O(len(s1) + len(s2))` | Создается новое множество, копируются элементы обоих исходных. |
| **Объединение на месте** | `s1.update(s2)` | `O(len(s2))` | `O(1)`* | Элементы `s2` добавляются в `s1`. *Может потребоваться `O(n)` памяти при рехэшировании. |
| **Пересечение** | `s1.intersection(s2)` | `O(min(len(s1), len(s2)))` | `O(min(len(s1), len(s2)))` | Обход меньшего множества для проверки наличия элементов в большем. |
| **Разность** | `s1.difference(s2)` | `O(len(s1))` | `O(len(s1))` | Обход `s1` и проверка отсутствия элементов в `s2`. |
| **Разность на месте** | `s1.difference_update(s2)` | `O(len(s1))` | `O(1)` | Удаление из `s1` элементов, присутствующих в `s2`. |
| **Симметричная разность** | `s1.symmetric_difference(s2)` | `O(len(s1) + len(s2))` | `O(len(s1) + len(s2))` | Объединение минус пересечение. |
| **Симм. разность на месте** | `s1.symmetric_difference_update(s2)` | `O(len(s1) + len(s2))` | `O(1)`* | Изменение `s1` без создания нового объекта. |

## Множество vs Замороженное множество (`frozenset`)

`frozenset` — это неизменяемая версия множества. Физически оно также использует разреженный массив индексации, что обеспечивает скорость поиска `O(1)`.

Благодаря неизменяемости, `frozenset` имеет ряд оптимизаций на уровне ядра CPython:

1. **Экономия памяти**:
   При создании размер массива индексации известен заранее. Не требуется выделять дополнительный буфер для будущих вставок, что снижает накладные расходы.

2. **Хэшируемость**:
   `frozenset` является хэшируемым объектом. Это позволяет использовать его в качестве ключа в словарях (`dict`) или элементов других множеств.

3. **Мгновенное копирование**:
   При копировании `frozenset` Python часто не создает глубокую копию данных, а просто возвращает ссылку на существующий объект, увеличивая счетчик ссылок.
   - Время: `O(1)`
   - Память: `O(1)`

4. **Free Lists (Списки свободных объектов)**:
   Уничтоженные объекты `frozenset` (содержащие до 80 элементов) не всегда сразу удаляются из памяти. Они кэшируются во внутренних списках свободных объектов для мгновенного переиспользования при создании новых множеств, что снижает нагрузку на аллокатор памяти.
