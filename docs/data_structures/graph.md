# Граф

**Граф** — это топологическая структура данных, состоящая из множества вершин (узлов) и множества ребер, соединяющих вершины. Каждый узел, с которым соединен некоторый узел, является ему соседом. Бывают случаи, когда узел соединен сам с собой — это называется **петлей**.

Для представления графов используется несколько способов:

- **Список смежности** — хэш-таблица, ключами которой являются вершины графа, а значениями — множества соседей.
  <details>

    ```python
    {
        "a": {"c", "d"},
        "b": {"c"},
        "c": {"a", "b"},
        "d": {"a"}
    }
    ```

  </details>
- **Матрица смежности** — двумерный массив, повторяющий структуру обычной таблицы. Наличие связи маркируется значением **1** в ячейке, а отсутствие связи — значением **0**.
  <details>

    ```python
    [
        #   a   b  c  d
        [0, 0, 1, 1],  # a
        [0, 0, 1, 0],  # b
        [1, 1, 0, 0],  # c
        [1, 0, 0, 0],  # d
    ]
    ```

  </details>

## Ключевые особенности

Основными ключевыми особенностями графа являются:

- **Ориентированность** — наличие направления у ребер. Если направление есть, то такой граф называется **ориентированным**. Если направление отсутствует, то такой граф называют **неориентированным**. Отсутствие направления можно рассматривать как простой признак наличия связи. Направление же задает жесткое отношение связи.
  <details>

    ```python
    # Ориентированный
    {
        "a": {"c"},  # направление связи a -> c
        "b": {"a"},  # направление связи b -> a
        "c": {"b"}   # направление связи c -> b
    }

    # Неориентированный
    {
        "a": {"c", "b"},
        "b": {"a", "c"},
        "c": {"b", "a"}
    }
    ```

  </details>
- **Взвешенность** — наличие веса у ребер. Если у ребер есть вес, то такой граф называют **взвешенным**. Если у ребер нет веса, то такой граф называется **невзвешенным**. Сам вес можно рассматривать как количественную оценку связи между вершинами.
  <details>

    ```python
    {
        "a": {"c": 0.5},  # вес связи a - 0.5 -> c
        "b": {"a": 1.0},  # вес связи b - 1.0 -> a
        "c": {"b": 0.2}   # вес связи c - 0.2 -> b
    }
    ```

  </details>
- **Цикличность** — наличие внутри графа замкнутых путей. Если в графе есть цикл, то такой граф называется **цикличным**. Если в графе нет циклов, то такой граф называют **ацикличным**. Петля является частным случаем цикла — это когда вершина замыкается сама на себе.
  <details>

    ```python
    {
        "a": {"c", "a"},  # есть петля a -> a
        "b": {"a"},       # путь замыкается a -> c -> b -> a
        "c": {"b"}
    }
    ```

  </details>

- **Путь** — последовательность вершин, в которой каждые две соседние вершины соединены ребром.
- **Связность** — свойство графа, при котором между любыми двумя вершинами существует путь. Граф называется связным, если он удовлетворяет этому условию.
- **Компонента связности** — максимальный связный подграф. В неориентированном графе все вершины разбиваются на непересекающиеся компоненты связности.
- **Степень вершины** — количество ребер, входящих и исходящих из вершины. Для неориентированного графа сумма степеней вершин **всегда** в два раза больше количества **ребер**. В неориентированном графе каждое ребро считается за 2 (входящее и исходящее). Вершины, у которых степень равна нулю, называются **изолированными**. Общая формула для расчета степени вершины:

  ```text
  deg(v) = deg⁺(v) + deg⁻(v)
  ```

## Сложность операций

При рассмотрении сложности операций над графом будем опираться на представление через **список смежности**.

*Формат: `Время | Пространство`*

| Операция | Временная сложность | Пространственная сложность | Примечание |
| --- | --- | --- | --- |
| **Добавление вершины** | `O(1)` | `O(1)` | Вставка нового ключа в хэш-таблицу. |
| **Удаление вершины** | `O(d)` | `O(1)` | Необходимо удалить все ссылки на вершину из списков соседей, где `d` — степень вершины. |
| **Добавление ребра** | `O(1)` | `O(1)` | Вставка элемента в множество соседей. |
| **Удаление ребра** | `O(1)` | `O(1)` | Удаление элемента из множества соседей. |
| **Проверка наличия ребра** | `O(1)` | `O(1)` | Проверка наличия элемента в множестве соседей. |
| **Получение соседей вершины** | `O(1)` | `O(1)` | Доступ к значению по ключу в хэш-таблице. |
| **Обход в ширину (BFS)** | `O(V + E)` | `O(V)` | Обход всех вершин `V` и ребер `E`. Пространство для очереди и множества посещенных. |
| **Обход в глубину (DFS)** | `O(V + E)` | `O(V)` | Обход всех вершин `V` и ребер `E`. Пространство для рекурсии или стека. |
| **Поиск кратчайшего пути (BFS)** | `O(V + E)` | `O(V)` | Для невзвешенных графов. |
| **Поиск кратчайшего пути (Дейкстра)** | `O((V + E) log V)` | `O(V)` | Для взвешенных графов с неотрицательными весами. |

## Список смежности vs Матрица смежности

Выбор представления графа зависит от задачи:

- Если граф **разреженный** (мало ребер), то правильный выбор — **список смежности**. Он экономит память и быстрее обходит соседей.
- Если граф **плотный** (много ребер), то лучше выбрать **матрицу смежности**. Проверка наличия ребра работает за `O(1)` без дополнительных структур.
- Если требуется **частая проверка наличия ребра**, то однозначный выбор — **матрица смежности**.
- Если требуется **обход графа**, то лучше выбрать **список смежности**, так как итерация по соседям быстрее.
