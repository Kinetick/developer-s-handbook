# Связанный список

**Связанный список** — это структура данных, призванная устранить недостатки [массива](./array.md#массив). Для связанного списка не требуется непрерывный блок памяти. Ему не требуется, чтобы все элементы были одного типа. Связанный список состоит из узлов, которые:

1. Хранят ссылку на значение в куче памяти.
2. Хранят ссылку на следующий узел.
3. Могут хранить ссылку на предыдущий узел.

---

## Ключевые особенности

В зависимости от направления связей внутри списка он может быть:

- **Односвязный список** — хранится только ссылка на **следующий** узел.
- **Двусвязный список** — хранятся ссылки на **следующий** и **предыдущий** узлы.

Связанный список может иметь одну критическую оптимизацию — ссылки на **голову** (самый первый узел) и **хвост** (самый последний узел). Эта оптимизация позволяет осуществлять мгновенную вставку/получение элементов в/из начала и конца. Именно такая гибкость позволяет использовать связанный список как **очередь** и **стек**.

---

## Реализация в CPython

В **Python** нет встроенной структуры данных — связанного списка. Наиболее приближенная к нему структура данных — это двунаправленная очередь (**deque**) из модуля `collections`. Важное замечание: в Python двунаправленная очередь всегда оптимизирована с головы и хвоста. Также очередь состоит из связанных блоков в виде массивов по 64 элемента.

Классическая реализация связанного списка может быть достигнута ручной разработкой:

<details>

```python
class Node:
    def __init__(
        self,
        key: str = "",
        value: t.Any = "",
        next_node: t.Self | None = None,
        prev_node: t.Self | None = None,
    ) -> None:
        self.key = key
        self.value = value
        self.next = next_node
        self.prev = prev_node

    def __str__(self) -> str:
        return f"key: {self.key}, value: {self.value}"


class DoubleLinkedList:
    def __init__(self) -> None:
        self._head = Node()
        self._tail = Node()
        self._head.next = self._tail
        self._tail.prev = self._head

    def insert_head(self, node: Node) -> None:
        first_next_node = self._head.next
        node.prev = self._head
        node.next = first_next_node

        if first_next_node:
            first_next_node.prev = node
        self._head.next = node

        return None

    def append_tail(self, node: Node) -> None:
        first_prev_node = self._tail.prev
        node.next = self._tail
        node.prev = first_prev_node

        if first_prev_node:
            first_prev_node.next = node
        self._tail.prev = node

        return None

    def remove_node(self, node: Node) -> None:
        if node.prev:
            node.prev.next = node.next
        if node.next:
            node.next.prev = node.prev
        node.next = node.prev = None

    def pop(self) -> Node | None:
        if self._tail.prev is self._head:
            return None

        result = self._tail.prev
        if result:
            self.remove_node(result)

        return result

    def popleft(self) -> Node | None:
        if self._head.next is self._tail:
            return None

        result = self._head.next
        if result:
            self.remove_node(result)

        return result
```

</details>

## Сложность операций

При рассмотрении сложности операций над связанным списком будем опираться на двунаправленную очередь (**deque**). И только отсутствующие операции рассмотрим в контексте двусвязного списка.

*Формат: `Время | Пространство`*

### Операции над deque

| Операция | Временная сложность | Пространственная сложность | Примечание |
| --- | --- | --- | --- |
| **Получение первого элемента** | `O(1)` | `O(1)` | Ссылка на первый элемент хранится в метаструктуре. |
| **Получение последнего элемента** | `O(1)` | `O(1)` | Ссылка на последний элемент хранится в метаструктуре. |
| **Добавление элемента в конец** | `O(1)` | `O(1)` | Получение последнего элемента связанного списка `O(1)`, обновление ссылок `O(1)`. |
| **Добавление элемента в начало** | `O(1)` | `O(1)` | Аналогично добавлению элемента в конец. |
| **Вставка элемента по индексу** | `O(n)` | `O(1)` | Во-первых, поиск нужного блока занимает `O(n)`. Вставка может быть мгновенной `O(1)`, если в блоке найдено место. Если свободного места нет, потребуется сдвиг блоков по цепочке. |
| **Разворот очереди** | `O(n)` | `O(1)` | Перестройка связей для всех блоков подразумевает физический обход за `O(n)`. |
| **Разворот очереди на k шагов** | `O(k)` | `O(1)` | Извлечение `k` элементов из хвоста очереди `O(k)`, вставка элементов в голову `O(k)`. |
| **Итерация по очереди** | `O(n)` | `O(1)` | Итерация по всем связанным блокам. |
| **Копирование очереди** | `O(n)` | `O(n)` | Итерация по связанному списку `O(n)`, копирование узлов, но не значений `O(n)`. |
| **Расширение очереди с концов** | `O(k)` | `O(k)` | Обход прицепляемой последовательности `O(k)`, перенос значений и расширение исходной очереди `O(k)`. |
| **Получение длины** | `O(1)` | `O(1)` | Значение длины всегда актуально и хранится в метаструктуре. |

### Операции над связанным списком

| Операция | Временная сложность | Пространственная сложность | Примечание |
| --- | --- | --- | --- |
| **Получение элемента по индексу** | `O(n)` | `O(1)` | Требуется обойти весь связанный список. |
| **Удаление элемента по индексу** | `O(n)` | `O(1)` | Требуется обойти весь связанный список для поиска удаляемого элемента `O(n)`. Сама операция удаления мгновенная `O(1)`. |
| **Расширение списка в середине** | `O(n + k)` | `O(k)` | Требуется итерация по связанному списку до места вставки `O(n)`, затем итерация по второму связанному списку `O(k)` с переносом узлов. Пространственная сложность возрастает до `O(k)`. |

## Связанный список (очередь) vs Массив (список)

Связанный список и массив решают совершенно разные задачи:

- Если требуется **чтение по индексу**, то правильный выбор — это **массив**.
- Если требуется **вставка в произвольное место**, то лучшим вариантом будет **связанный список**. Да, та же самая очередь делает вставку в произвольное место за `O(n)`, но при этом не требуется перемещения объектов массива. Сама операция вставки мгновенная **O(1)**.
- Если требуется **забирать элементы с конца** (стек), то лучше выбрать **массив**, здесь все операции будут `O(1)`.
- Если требуется **забирать элементы с начала**, то однозначный выбор — это **очередь**.
